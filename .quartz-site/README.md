# Публикация книги правил

Этот каталог хранит техническую обвязку для публикации книги правил через Quartz.

Источник для сайта только один:

- `../Книга правил/Проект Андромеда v0.4.md`

Перед сборкой Quartz автоматически копирует этот файл в `content/index.md`.

`content/index.md` не редактируется вручную. Это служебная копия для Quartz, а канонический исходник все равно остается в `../Книга правил/Проект Андромеда v0.4.md`.

## Команды

- `npm ci` — установить зависимости.
- `npm run build` — синхронизировать книгу и собрать сайт.
- `npm run dev` — локальный предпросмотр с автосинхронизацией, если меняется файл книги.

## GitHub Pages

Workflow лежит в `../.github/workflows/deploy-rulebook.yml`.

После первого пуша нужно один раз открыть репозиторий на GitHub:

1. `Settings`
2. `Pages`
3. В `Source` выбрать `GitHub Actions`

После этого сайт будет обновляться автоматически при каждом пуше в `master` или `main`, если изменился сам файл книги, Quartz-конфиг или workflow.
